# 算法性能分析

### 时间复杂度

渐进时间复杂度，简称时间复杂度

O( ) 学术来讲指算法复杂度上界，但通常业内是指一种算法的一般情况省去常数项系数是因为大O的根本定义，即数量级突破一个点且数据量巨大

O(1)常数项<O(logn)对数阶<O(n)线性阶<O(nlogn)线性对数阶<O(n^2)平方阶<O(n^3)立方阶<O(2^n)指数阶

O(logn)的底数可以是任何数，回忆为何能把底数忽略！提示：根本原因是底数的多少对O( )级别的分析不产生很大影响

例题：从n个字符串中找到相同的两个字符串（假设这里只有两个相同的字符串

### 超时问题

估算CPU 1s内处理问题规模：

| 算法时间复杂度 | 1s内可以处理的大概n的规模 |
| :------------: | :-----------------------: |
|      O(n)      |        常数项*10^8        |
|     O(n^2)     |        常数项*10^4        |
|    O(nlogn)    |        常数项*10^7        |
